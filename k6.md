K6 stepped RPM test

Files added:
- k6-login.mjs — k6 script that POSTS the login payload.
- k6-visits-by-day.mjs — k6 script that GETs the visits-by-day report.
- k6-visits-by-doctor.mjs — k6 script that GETs the visits-by-doctor report.
- run_k6_steps.ps1 — PowerShell runner to step RPM and stop when failures exceed threshold.
- parse_k6_summary.py — parses k6 summary JSON and returns non-zero when failure rate > threshold.

Quick usage (PowerShell):

1. Ensure `k6` is installed and on PATH.

2. Run the step test (defaults: start 100 RPM, step 100, max 2000 RPM, 1m per step, failure threshold 1%):

```powershell
.\run_k6_steps.ps1 -StartRPM 100 -Step 100 -MaxRPM 2000 -Duration '1m' -FailureThresholdPercent 1.0
```

3. To test the visits-by-day endpoint, pass the dedicated script and a result prefix:

```powershell
.\run_k6_steps.ps1 -ScriptPath 'k6-visits-by-day.mjs' -OutputPrefix 'k6-visits-by-day-rpm' -StartRPM 100 -Step 100 -MaxRPM 2000 -Duration '1m' -FailureThresholdPercent 1.0
```

	The same pattern works for visits-by-doctor:

```powershell
.\run_k6_steps.ps1 -ScriptPath 'k6-visits-by-doctor.mjs' -OutputPrefix 'k6-visits-by-doctor-rpm' -StartRPM 100 -Step 100 -MaxRPM 2000 -Duration '1m' -FailureThresholdPercent 1.0
```

	The visits-by-day script requires `AUTH_TOKEN` from a fresh login. Example:

```powershell
$body = @{ email = 'neeraj.singh@apexnovapharma.com'; password = 'Mr@123' } | ConvertTo-Json
$login = Invoke-RestMethod -Uri 'https://staging.fieldtrix.com/api-staging/core/auth/login' -Method Post -Headers @{ Accept = 'application/json' } -ContentType 'application/json' -Body $body
$env:AUTH_TOKEN = $login.data.access_token
```

4. To change credentials or base URL, pass env vars to k6 via the runner or edit script environment variables. Example (manual k6 run):

```powershell
$env:EMAIL='user@example.com'; $env:PASSWORD='secret'; $env:RPM=100; k6 run --summary-export=results/k6-rpm-100.json k6-login.mjs
```

Results are saved to the `results/` directory.
